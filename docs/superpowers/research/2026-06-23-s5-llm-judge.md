# S5 — LLM-judge (retrieve → narrate)

**Date:** 2026-06-23
**Module:** `integritybg/judge.py`. Model: Gemini (`gemini-3.1-flash-lite-preview`, Gemma fallbacks) via
`google-genai`, reusing dokoga's `GOOGLE_API_KEY`. Data sent is public CC0 only.

## Design — the LLM explains, the deterministic layer decides

Strict **retrieve → narrate** (dokoga's grounding pattern):
1. **retrieve** (deterministic SQL): assemble the exact evidence — contract facts, every flag + its
   evidence JSON, the Integrity Index level/score, the ML `p_adverse`, and the blended risk.
2. **narrate** (LLM): explain it in plain Bulgarian under a hard grounding contract —
   *use only the given numbers, compute nothing, this is a risk SIGNAL not an accusation, end with a
   disclaimer.* The model never produces a score and cannot hallucinate sums.

Both narratives: use only supplied facts, invent no numbers, factual (not accusatory), end with the
mandatory disclaimer. Exactly the honest-by-construction behaviour the project requires.

## Reproduce

